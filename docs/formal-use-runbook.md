# 正式功能使用运行手册

这份手册用于把当前版本交给内部产品、研发和市场同事正式使用。当前使用目标是验证清洁电器竞品分析工作台能否支撑真实资料导入、人工复核、型号对比、路线图导出和交接包归档；不包含公开 SaaS、多用户账号、计费或完整 Next.js/Prisma 迁移。

## 使用目标

- 用真实扫地机、洗地机、吸尘器资料完成一次端到端竞品分析。
- 验证 URL、长图、PDF、手工录入和 CSV 导入是否能进入同一产品库。
- 验证 500 字以内型号对比总结是否覆盖产品功能、关键参数、使用感受、价格梯度和 Top3 卖点。
- 验证品牌路线图、产品库、对比表、审计日志、AI 用量和完整数据包都能导出。
- 在正式功能使用包中记录 UI 字号、模块密度、字段口径、AI 低置信复核和导出格式的真实反馈，网页内不增加单独试用模块。

## 参与角色

- 使用负责人：启动服务、准备访问令牌、收集反馈和确认 Go/No-Go。
- 产品研究员：导入真实资料、补充字段、确认卖点和低置信项。
- 产品经理：检查型号对比总结、路线图和字段配置是否符合决策口径。
- 技术负责人：查看用量日志、错误、备份、数据包和发布验收结果。

## 使用前置条件

1. 拉取最新代码后运行 `node scripts/verify-release.mjs`，必须通过。
2. 复制 `.env.example` 为 `.env.local`，至少确认 `PORT`、访问令牌、`AI_PROVIDER=deepseek`、`VISION_PROVIDER=qwen`、`DEEPSEEK_API_KEY` 和 `QWEN_API_KEY`。
3. 当前 DeepSeek 负责文本型结构化抽取和总结；Qwen-VL 负责图片、长图和 URL 图片候选识别；原始 PDF 未转图片时进入人工复核兜底，不做虚假视觉识别。
4. 运行 `node scripts/check-local-env.mjs`，确认 `data/` 可写、端口空闲或当前端口返回工作台首页。
5. 启动 `node server.mjs`，打开 `http://localhost:4173`。
6. 导出一次完整数据包作为正式使用前基线；正式导入数据包前系统会自动下载备份。

## 使用数据建议

- 每个品类至少 2 个型号：扫地机、洗地机、吸尘器都要覆盖。
- 至少包含 2 条 URL、1 张详情页长图、1 个 PDF 和 1 次 CSV 批量导入。
- 产品图优先使用详情页或用户上传资料中的真实图片，不生成虚构产品图。
- 电商和官网抓取只使用允许访问的公开页面；遇到登录、验证码或反爬限制时改用截图或 PDF。

## 使用任务

1. 筛选工作台：组合关键词、品类、价格段、品牌、渠道、上市状态和自定义功能参数，保存并恢复一个工作视图。
2. 自定义字段：新增一个枚举字段和一个布尔字段，录入到产品详情，确认它们进入筛选器、对比矩阵和导出列。
3. AI 导入：分别用 URL、长图和 PDF 导入资料，检查页面证据包、价格候选、图片候选、字段置信度和待确认队列。
4. 型号对比：选择 2-5 个型号，勾选关键功能字段，生成 500 字以内总结并导出对比表。
5. 路线图：按品牌、品类、状态和季度筛选，导出路线图 Excel、SVG 和各品牌分页 PDF。
6. 数据包：导出完整 JSON 数据包，再导入同一数据包，确认导入前备份、审计日志和保存视图存在。
7. 用量与审计：导出 AI 用量 CSV 和审计 CSV，确认 provider、model、status、usage、estimatedCostUsd 和关键操作可追溯。
8. 数据包与审计导出：导出完整 JSON 数据包、AI 用量 CSV 和审计 CSV，确认数据可备份、可恢复、可追溯。
9. 正式功能使用反馈：在 `reports/formal-use-pack-*.md` 的反馈记录区填写至少 1 条问题或改进建议，并给出 Go/No-Go 建议。

## Go/No-Go 标准

Go：

- `node scripts/verify-release.mjs` 通过。
- 至少 1 轮人工浏览器冒烟通过，覆盖筛选、自定义字段、AI 导入兜底、型号对比、路线图、数据包、审计和用量。
- 真实样例覆盖扫地机、洗地机、吸尘器，以及 URL、长图、PDF。
- 正式使用期间没有数据丢失、密钥泄露、导出阻塞或无法恢复的问题。

No-Go：

- 发布验收失败。
- 数据包导入导出或备份失败。
- AI 失败时无法进入人工复核兜底。
- 导出文件缺少产品图、价格、Top3 卖点或关键字段。
- 控制台出现影响主流程的 error/pageerror。

## 回滚与备份

- 代码回滚：使用上一个通过 `node scripts/verify-release.mjs` 的 Git 提交。
- 数据回滚：优先导入 `backup-before-import` JSON；没有备份时使用正式使用前导出的完整数据包。
- 配置回滚：恢复 `.env.local` 中 `PORT`、访问令牌和模型 provider 配置，不能提交真实密钥。
- 报告归档：保留 `reports/mvp-test-report-*.md`、`reports/manual-smoke-checklist-*.md` 和 `reports/formal-use-pack-*.md` 作为正式使用证据。

## 不做事项

- 不公开注册，不做多租户 SaaS。
- 不绕过登录、付费墙、验证码或平台反爬限制。
- 不把真实 API key、`data/` 本地数据和 `reports/` 产物提交到 Git。
- 不在正式使用准备阶段一次性迁移到 Next.js/Prisma；正式化迁移仍按现有路线分阶段推进。
